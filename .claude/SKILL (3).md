## 3. 취약점 분류 (빠른 참조)

자세한 내용 → `references/vuln-types.md`

### 주요 카테고리
- **인젝션**: SQL·Command·LDAP Injection
- **인증·세션**: 취약한 인증, 세션 고정
- **접근 제어**: 수평·수직 권한 상승
- **암호화**: 약한 알고리즘, 하드코딩 키
- **설정 오류**: 기본 계정, 불필요한 서비스
- **의존성**: 취약한 서드파티 라이브러리

## 4. CVSS 심각도 기준

자세한 채점 방법 → `references/cvss-guide.md`

| 점수 범위 | 등급 | 대응 우선순위 |
|-----------|------|--------------|
| 9.0 – 10.0 | Critical | 즉시 패치 (24h 이내) |
| 7.0 – 8.9  | High     | 긴급 패치 (7일 이내) |
| 4.0 – 6.9  | Medium   | 계획 패치 (30일 이내) |
| 0.1 – 3.9  | Low      | 정기 패치 주기 반영 |

## 5. 코드 보안 리뷰 체크리스트

### 공통 점검 항목
- [ ] 입력값 검증 (화이트리스트 방식 우선)
- [ ] 출력 인코딩 (XSS 방지)
- [ ] 파라미터화된 쿼리 사용 여부
- [ ] 인증·인가 로직 일관성
- [ ] 민감 정보 하드코딩 여부 (API 키, 비밀번호)
- [ ] 에러 메시지 정보 노출
- [ ] 암호화 알고리즘 적절성 (MD5·SHA1 사용 여부)
- [ ] 의존성 패키지 최신 버전 여부

언어별 세부 체크리스트 → `references/vuln-types.md#언어별-패턴`

## 6. 리포트 작성 규칙

전체 템플릿 → `references/report-template.md`

### 취약점 항목 필수 포함 요소
1. **제목**: 취약점 유형 + 위치 (예: `[SQL Injection] /api/login - username 파라미터`)
2. **심각도**: CVSS 점수 + 등급
3. **설명**: 취약점 원인 및 동작 방식 (2–4문장)
4. **재현 방법 (PoC)**: 단계별 재현 절차
5. **영향**: 기밀성·무결성·가용성 영향
6. **권고 사항**: 구체적 수정 코드 또는 설정 변경

---

## 7. 윤리 및 법적 주의사항

> ⚠️ **중요**: 이 스킬은 방어 목적의 보안 분석에만 사용.

- 실제 공격 코드(익스플로잇) 작성은 지원하지 않음
- 권한 없는 시스템 대상 분석 코드 제공 금지
- 취약점 정보는 책임감 있는 공개(Responsible Disclosure) 원칙 준수
- 법적 근거 없는 침투 테스트 조력 불가
